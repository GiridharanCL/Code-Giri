ESP32 HTTP POST (IFTTT):

Construct a model asynchronously from a compiled model asset.

ESP32 makes an HTTP POST request to send a new value to IFTTT.


HTTP POST :


The Hypertext Transfer Protocol (HTTP) works as a request-response protocol between a server and a client. It enables smooth communication between them. The two most widely used HTTP methods are GET and POST. Through GET, data is requested from a specified resource whereas POST sends data to create a resource. In this user guide, we will focus on the POST request in HTTP. It is used to send information from the client to the server to create or update a resource. This information is included in the request body of the HTTP request.

HTTP POST Request Method :


The Hypertext Transfer Protocol (HTTP) works as a request-response protocol between a client and server. Here’s an example:
▪ The ESP32 (client) submits an HTTP request to a Server
▪ The server returns a response to the ESP32 (client)
▪ Finally, the response contains status information about the request and may also contain the requested content.

HTTP POST :


POST is used to send data to a server to create/update a resource. For example, publish sensor readings to a server. The data sent to the server with POST is stored in the request body of the HTTP request:
POST /update HTTP/1.1
Host: example.com
api_key=api&field1=value1
Content-Type: application/x-www-form-urlencoded
In the body request, you can also send a JSON object:
POST /update HTTP/1.1
Host: example.com
{api_key: "api", field1: value1}
Content-Type: application/json

ESP32 HTTP POST in IFTTT :


Send HTTP POST data in ESP32 to IFFT. In return, it will trigger the IFTTT API to send us email notifications regarding our data. IFTTT means ‘If this, then that.’ It is an open-source service that gives the user the freedom to program a response to an event according to their likes. Users can create an applet which are chains of conditional statements by a combination of several app services and add triggering parameters. Whenever ESP32 will make a request to the server, the IFTTT applet will send three random values to email. In practical situations, these values can be readings from sensors. To work with this web service, users will have to follow a series of steps to ensure the proper functionality.


IFTTT :


Although the IFTTT service is free to use, we will have to create an account. First go to the following website: https://ifttt.com/ The following window will appear. Click on the ‘Get Started’ icon.


The following window will appear. Users can ‘sign up’ with email account.


Click the ‘sign up’ tag. Then enter the email address and password to Get started for working in IFTTT. This whole process is free of cost for the first three applets.


Creating an Applet:


After creating the account, users will be directed to the page where they will create an applet. Click on ‘Create.’


The next window opens up. Click the following Add icon in the ‘If This’ section.


Another page will open in which users have to choose the service. There are a lot of options to choose from. Write down ‘webhooks’ in the search option and its icon will appear,


Next, choose the trigger as: ‘Receive a web request’ by clicking on it. Whenever webhooks receive a web request, some action will take place. This we will define in the ‘THAT’ section.


After clicking the Receive a web request, the following window will open up. We will write down Readings as the event name for the web request. Use any other name of user choice. Click the ‘Create Trigger’ icon.


After the trigger is created, then taken back to the web page where first added the service for the ‘IF THIS’ section. Now click the ADD icon for the ‘THEN THAT’ section.


Select and Setup the Service :


Now choose the service. Users have to choose what will happen if a web request is received. Thus, we will type ‘Email’ in the search option and click on its icon. This is because it sends an email whenever a POST request is received.


The next window opens up. Click on the ‘Send me an email’ icon.


Fill in all the entries as given below and then click on the ‘Create Action’ icon.


After creating the action, then back to the initial web page of IFTTT. Click ‘Continue’ to proceed.


After this click the ‘Finish’ icon.


You have successfully created the applet as shown below as ‘connected’.


Testing the Applet :


Go to your and select “My Services” or open a webpage with the link: https://ifttt.com/my_services The following windows will appear. Afterwards, click on Webhooks.


This will take users to the next web page. Click on ‘Documentation’ icon.


User will receive a key which should be secure. Next, enter the details as shown to trigger the event. Then give hypothetical values for the three variables for now. At the end click, ‘Test it’ icon.


Then get a notification on top of the web page as the Event has been triggered. Now go to the email account and check for any notification that came from the IFTTT service. Open the message and it will display the dummy values as entered before.


If you also received a message with the same readings that entered, then it means that applet is running successfully.

Loading a Model

Use this method to load a model asynchronously. Core ML calls your completion handler after it successfully loads the model, or encounters an error attempting to load it.


[MLModel loadContentsOfURL:modelURL
configuration:[[MLModelConfiguration alloc] init]
completionHandler:^(MLModel *loadedModel,
NSError *error) {
if (nil == loadedModel) {
NSLog ( @"Error loading model`: %@" , ( nil != error) ? error : modelURL);
return ;
}
NSLog ( @"Successfully loaded model at: %@" , modelURL);
// Use the loaded model for predictions.// ... }];


If the model loaded successfully, you can use the MLModel instance in your completion hander; otherwise, use the NSError instance to address the error. See MLModelError for the list of error codes.
