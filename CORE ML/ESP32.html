In this project, create a standalone web server with an ESP32 that controls outputs (three LEDs) using the Arduino IDE programming environment. The web server is mobile responsive and can be accessed with any device that has a browser on the local network.
Outline:
· The web server controls three LEDs connected to the ESP32 25, 26 and 27.
· The ESP32 web server accesses the ESP32 by typing the ESP32 IP address on a browser in the local network.
· By clicking the buttons on the web server can instantly change the state of each LED.
· This is just a simple example to illustrate how to build a web server that controls outputs, the idea is to replace those LEDs with a relay, or any other electronic components required.
Arduino IDE that allows programming the ESP32 using the Arduino IDE and its programming language.



Required Components:


1· ESP32 development board
2· 3 x LED
3· 3 x Resistor
4· Breadboard
5· Jumper wires

Uploading the Code :


Now, upload the code and the web server will work straight away. Follow the next steps to upload code to the ESP32. Plug the ESP32 board in the computer.
* In the Arduino IDE select the board in Tools > Board (in our case we’re using the ESP32 DEVKIT DOIT board).
* Select the COM port in Tools > Port.
* Press the Upload button in the Arduino IDE and wait a few seconds while the code compiles and uploads to the board.
* Wait for the “Done uploading” message.
* After uploading the code, open the Serial Monitor at a baud rate of 115200.
* Press the ESP32 EN button (reset). The ESP32 connects to Wi-Fi, and outputs the ESP IP address on the Serial Monitor.
* Copy that IP address, because it needs to access the ESP32 web server.



To access the web server, open the browser, paste the ESP32 IP address.

In Serial Monitor, ESP receives an HTTP request from a new client (in this case, this browser). At the same time, in Serial Monitor when clicking the button to turn the 25th Pin ON, ESP32 receives a request on the /25/on URL, and this also repeats on the 26th Pin and 27th Pin.


Display the on or the off button, depending on the current state of the LED 1. If the current state of the LED1 is off, it shows the ON button, if not, it displays the OFF button.

Loading a Model

Use this method to load a model asynchronously. Core ML calls your completion handler after it successfully loads the model, or encounters an error attempting to load it.


[MLModel loadContentsOfURL:modelURL
configuration:[[MLModelConfiguration alloc] init]
completionHandler:^(MLModel *loadedModel,
NSError *error) {
if (nil == loadedModel) {
NSLog ( @"Error loading model`: %@" , ( nil != error) ? error : modelURL);
return ;
}
NSLog ( @"Successfully loaded model at: %@" , modelURL);
// Use the loaded model for predictions.// ... }];


If the model loaded successfully, you can use the MLModel instance in your completion hander; otherwise, use the NSError instance to address the error. See MLModelError for the list of error codes.
